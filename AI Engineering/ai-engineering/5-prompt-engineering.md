# 5. 프롬프트 엔지니어링

- 프롬프트 엔지니어링: 모델이 원하는 결과를 생성하도록 지시를 다듬는, 가장 쉽고 일반적인 **모델 조정(model adaptation)** 기법
- 파인튜닝과 달리 모델의 가중치를 변경하지 않고도 모델의 응답을 조정
- 따라서 파인튜닝 같이 더 많은 자원이 필요한 기법으로 넘어가기 전에 프롬프팅을 최대한 활용해 보는 것이 좋음
- 프롬프트 엔지니어링은 별것 아니라고 오해받기 쉬움
  - 하지만 문제를 해결하기 위한 창의적인 접근법이 될 수 있음
  - 체계적인 실험 방법과 평가 과정을 통해 다른 ML 실험처럼 엄중히 수행될 수 있음
- 프롬프트 엔지니어링 = 사람 ↔ AI 커뮤니케이션
- 5장에서는 효과적인 프롬프트 작성법, 프롬프트 공격 방어법을 다룰 것

## 5.1 프롬프트 소개

- 프롬프트가 일반적으로 포함하는 요소들
  - **작업 설명**: 모델이 수행해야 할 일, 역할 및 출력 형식 포함
  - **작업 수행 방법 및 예시**: 모델이 작업을 수행하는 방법 및 예시
  - **작업**: 모델이 수행해야 할 구체적인 작업
- 프롬프트 엔지니어링의 필요도는 모델이 프롬프트 변화에 얼마나 **강건하게(robust)** 반응하는지에 따라 달라짐
  - 프롬프트가 약간만 바뀌어도 응답이 완전히 달라진다면 프롬프트 엔지니어링의 필요성이 커짐
  - 프롬프트를 무작위로 변경하면서 출력의 변경을 확인하면 모델의 강건성 측정 가능
  - 모델이 더 강력해질수록, 모델이 더 강건해짐

### 인컨텍스트 학습(In-Context Learning): 제로샷과 퓨샷

- **인컨텍스트 학습(In-Context Learning, ICL)**
  - 프롬프트를 통해 모델에게 무엇을 해야 할지 가르치는 것
  - 이 방법을 통해 모델은 원래 학습 목적과 다른 작업이더라도 프롬프트 내 행동을 통해 학습할 수 있음
  - 모델이 지속적으로 새로운 정보를 학습할 수 있게 해주므로, 모델이 계속 발전할 수 있게 만들어줌
- **Shot**
  - 프롬프트에 제공된 각 예시를 샷(shot)이라고 부름
  - 모델에게 프롬프트의 예시들을 통해 학습하도록 가르치는 방식을 **퓨샷(few-shot)** 학습이라고 부름
  - 만약 5개의 예시가 있다면 5-shot 학습
  - 만약 예시가 전혀 제공되지 않으면 **제로샷(zero-shot)** 학습
  - 실험을 통해 애플리케이션에 필요한 적정 샷 갯수를 찾아야 함
  - 샷 갯수는 모델의 최대 컨텍스트 길이에 의해 제한되며, 샷 갯수가 많을수록 프롬프트가 길어져 추론 비용이 증가
- 용어 모호성: 프롬프트와 컨텍스트
  - 아직도 용어 통합이 안 되어 있으며 논의가 이어지고 있음
  - 프롬프트: 모델에 입력되는 전체 내용
  - 컨텍스트: 모델이 주어진 작업을 수행할 수 있도록 모델에 제공되는 정보
- 프롬프트 엔지니어링은 원하는 프로그램을 활성화할 수 있는 적절한 프롬프트를 찾는 과정이라고 할 수 있음

### 시스템 프롬프트와 사용자 프롬프트

- 대부분의 모델 API는 프롬프트를 시스템 프롬프트와 사용자 프롬프트로 나눌 수 있는 옵션 제공
- 시스템 프롬프트: 작업 설명
- 사용자 프롬프트: 작업 자체
- ChatGPT를 비롯한 거의 모든 생성형 AI 애플리케이션에는 시스템 프롬프트가 존재
- 일반적으로 애플리케이션 개발자가 제공하는 지시는 시스템 프롬프트로, 사용자가 제공하는 지시는 사용자 프롬프트로 구성됨
- 시스템 프롬프트와 사용자 프롬프트가 주어지면 모델은 일반적으로 템플릿에 맞게 하나의 프롬프트로 결합
- 모델별로 서로 다른 채팅 템플릿을 사용하며, 같은 모델이라도 버전에 따라 템플릿이 달라질 수 있음
- 시스템 프롬프트가 사용자 프롬프트보다 성능을 더 향상시키는 경향이 있음
  - 시스템 프롬프트가 최종 프롬프트의 맨 앞에 위치하기 때문에
  - 모델이 시스템 프롬프트를 더 중요하게 인식하도록 학습되었을 수 있음

### 컨텍스트 길이와 컨텍스트 효율성

- 모델 제공업체들은 경쟁적으로 컨텍스트 길이 확장을 해오고 있으며, Gemini-1.5 Pro는 2M(200만) 토큰의 컨텍스트 길이를 지원
- 연구에 따르면 모델은 프롬프트의 중간보다 시작과 끝에 제시된 지시를 더 잘 이해함
- **NIAH(Needle-in-a-Haystack)** 테스트
  - 프롬프트의 다양한 부분에 대한 효과를 평가하는 방법
  - 무작위 정보(바늘)를 프롬프트(건초더미)의 다양한 위치에 삽입하고 모델에게 이를 찾도록 요청
  - Liu 등의 연구 결과, 모델들이 중간보다 시작과 끝에 가까울 때 정보를 훨씬 더 잘 찾아내는 것으로 나타났음
- 만약 모델 성능이 컨텍스트 길이가 늘어남에 따라 더 나빠진다면, 프롬프트를 더 간결하게 만드는 방법을 고려해야 함
- 프롬프트의 핵심 구성 요소들
  - 시스템 프롬프트
  - 사용자 프롬프트
  - 예시
  - 컨텍스트
