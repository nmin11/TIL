# 7. 파인튜닝

- 파인튜닝: 모델 전체나 일부를 추가로 학습시켜 특정 작업에 맞게 모델을 조정하는 과정
- 파인튜닝은 모델의 가중치를 조정하는 방식으로 모델을 변화시킴
- 파인튜닝을 통해 모델의 다양한 측면을 향상시킬 수 있음
  - 도메인별 능력
  - 안전성 강화
  - 하지만 무엇보다 많이 사용되는 목적은 모델의 지시 수행 능력 향상
- **"언제 파인튜닝을 하고 언제 RAG을 해야 하는가?"**
  - 7장에서는 파인튜닝을 해야 하는 이유와 그렇지 않은 이유를 살펴볼 것
  - 더 나아가 파인튜닝과 다른 방법 중 무엇을 선택할 때 도움이 되는 프레임워크도 소개할 것
- 파인튜닝은 프롬프트 기반 방법에 비해 훨씬 많은 메모리가 필요
  - **PEFT(Parameter-Efficient Fine-Tuning)** : 메모리 효율성을 높이는 파인튜닝 기법
- 파인튜닝은 모델을 직접 학습시키는 과정이므로 ML 지식이 필요
  - 이 책에서는 ML 기초를 전부 다루기 보다는 몇 가지 핵심 개념만 다룰 것

## 7.1 파인튜닝 개요

- 파인튜닝은 기본적인 능력을 갖춘 기본 모델을 토대로 시작
- 파인튜닝의 목표: 모델이 특정 작업을 충분히 잘 수행하도록 만드는 것
- 파인 튜닝은 **전이 학습(Transfer Learning, TL)** 의 한 형태
  - 전이 학습: 한 과제에서 학습한 지식을 다른 과제에 적용하는 방법
  - 딥러닝 초기부터 전이 학습은 학습 데이터를 구하기 어려울 때 좋은 해결책이 되어 왔음
  - 전이 학습은 **표본 효율성(sample efficiency)** 을 높여 모델이 더 적은 예시로도 같은 행동을 학습할 수 있게 함
  - 전이 학습 외에 또 다른 접근법으로는 **특성 기반 전이(feature-based transfer)** 가 있음
    - 모델이 학습한 지식을 고정한 채, 모델에서 Feature를 추출하고, 이 Feature를 기반으로 새로운 모델을 학습시키는 방식
- 이상적인 파인튜닝은 모델이 이미 학습할 것들을 학습한 상태에서 단지 모델의 행동을 다듬는 과정임
- 파인튜닝은 모델 학습 과정의 일부로, 사전 학습의 확장이라고 볼 수 있음
- 모델 학습은 보통 자기 지도 학습 방식의 사전 학습에서 시작됨
  - 모델은 자기 지도 학습을 통해 레이블 없는 대량의 데이터로부터 학습
  - 비싼 작업별 데이터로 사전 학습 모델을 파인튜닝하기 전에, 저렴한 관련 분야 데이터로 자기 지도 학습을 적용해볼 수 있음
  - 이러한 자기 지도 파인튜닝을 **지속적 사전 학습(Continual Pretraining)** 이라고 부름
- **Infilling finetuning**
  - 지도 파인튜닝을 통해 다음 토큰을 예측하거나 빈칸을 채우도록 모델을 조정하는 방식
  - 자기회귀 방식으로 사전 학습된 모델이라도 infilling finetuning을 적용할 수 있음
    - 바로 다음 토큰만 예측하는 것이 아닌, 빈칸을 채우는 것처럼 학습시키는 것이 가능하다는 뜻
- 자기 지도 학습 이후
  - 자기 지도 학습을 통해 모델은 방대한 지식을 얻지만, 사용자가 원하는 작업에 바로 활용하기는 어려움
  - 지도 파인튜닝을 통해, 모델과 사람의 실제 선호와의 격차를 줄여야 함
- 지도 파인튜닝
  - 고품질 주석 데이터를 활용해 모델을 사람의 사용 방식과 선호도에 맞게 조정하는 과정
  - (입력, 출력) 쌍으로 모델을 학습시킴
  - 입력은 지시가 될 수 있으며, 출력은 응답이 될 수 있음
  - 응답은 책 요약처럼 개방형 형태일 수도 있고, 분류 작업처럼 폐쇄형 형태일 수도 있음
- 선호도 파인튜닝
  - 강화 학습을 통해 사람의 선호도를 최대화하는 응답을 생성하도록 조정하는 과정
  - 주로 (지시, 선호 응답, 비선호 응답) 형식의 비교 데이터가 필요
- **Long-context finetuning**
  - 컨텍스트 길이를 늘리는 파인튜닝
  - 보통 위치 임베딩 조정 같은 모델 구조의 수정이 필요
  - 롱 시퀀스는 토큰의 가능한 위치가 많다는 뜻이며, 이를 위치 임베딩이 처리할 수 있어야 하므로
  - 다른 파인튜닝 기법보다 까다로운 방법
  - 결과로 얻어진 모델은 숏 시퀀스에서 성능이 오히려 떨어질 수 있음
- 파인튜닝은 모델 개발자와 애플리케이션 개발자 모두 수행 가능
  - 모델 개발자: 주로 모델 출시 전 다양한 파인튜닝 기법으로 모델을 사후 학습시킴
  - 애플리케이션 개발자: 대개 이미 사후 학습된 모델을 더 정교하게 파인튜닝
