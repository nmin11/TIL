## 멀티태스킹

- 하나의 CPU 코어로 여러 프로그램을 동시에 실행하는 기술
- 마치 동시에 실행되는 것처럼 하는 기법 - 시분할(Time Sharing) 기법
- CPU가 어떤 프로그램을 얼만큼 실행할지는 운영체제가 결정
  - 스케줄링이라고 부름
- **운영체제 소프트웨어의 관점!**
  - 소프트웨어 기반으로 CPU 시간을 분할하여 각 작업에 할당한다

## 멀티프로세싱

- 컴퓨터 시스템에서 둘 이상의 프로세서(CPU 코어)를 사용해서 여러 작업을 동시에 처리하는 기술
- 현대에는 하나의 CPU가 여러 개의 코어를 가짐
- **하드웨어 장비의 관점!**
  - 하드웨어 기반으로 성능을 향상시킨다

## 프로세스

- **운영체제 안에서 실행중인 프로그램이 프로세스**
- Java 언어로 비유하자면,
  - 클래스는 프로그램
  - 인스턴스는 프로세스
- 격리되어 독립된 메모리 공간을 가진다
  - 다른 프로세스에 영향을 주지 않음

### 프로세스의 메모리 구성

- 코드 섹션: 프로그램 코드 저장
- 데이터 섹션: 전역 변수 및 정적 변수 저장
- 힙: 동적 메모리 영역
- 스택: 메서드 호출 시 생성되는 지역 변수 및 반환 주소 저장 (스레드에 포함됨)

## 스레드

- **프로세스는 하나 이상의 스레드를 반드시 포함한다**
- 스레드는 프로세스 내에서 실행되는 작업 단위
- 스레드들은 프로세스의 메모리 공간을 공유
- 프로세스의 코드는 반드시 스레드에 의해 한 줄 씩 실행되어야 하는 것

### 스레드의 메모리 구성

- 공유 메모리: 프로세스의 코드 섹션, 데이터 섹션, 힙은 공유됨
- 개별 스택: 각 스레드는 자신만의 스택을 가짐

⇒ 프로세스는 실행 환경과 자원을 제공하는 컨테이너, 스레드는 CPU를 사용해서 코드를 하나씩 실행

## 스레드와 스케줄링

- 운영체제는 내부에 스케줄링 큐를 가지고 있고, 각각의 스레드는 스케줄링 큐에서 대기
- CPU 코어들은 스케줄링 큐에서 스레드의 작업들을 폴링하면서 실행을 반복
- 운영체제의 스케줄링은 CPU를 최대한 활용할 수 있도록 다양한 우선순위 및 최적화 기법을 사용

## 컨텍스트 스위칭

- 스레드 A를 멈추고 스레드 B로 가기 위해서는 스레드 A가 멈춘 시점에 사용하던 CPU 관련 값들을 메모리에 저장해두어야 함
- 이후 다시 멈춰둔 스레드를 재시작하려면 저장해둔 값을 다시 불러와야 함
- 약간의 비용이 발생하는 작업
- **멀티스레드는 대부분 효율적이지만, 컨텍스트 스위칭이 필요하므로 항상 효율적이진 않다**
- 스레드에 비해 CPU 코어 개수가 극단적으로 작은 경우 컨텍스트 스위칭 방식이 불리할 것
  - 실제 컨텍스트 스위칭 시간은 매우 짧기 때문
  - 이상적으로는 `스레드 수 = CPU 코어 수 + 1`

### CPU 바운드 작업

- CPU 연산 능력을 많이 요구하는 스레드 작업
- 주로 계산, 데이터 처리, 알고리즘 실행 등 CPU 처리 속도가 작업 완료 시간을 결정하는 경우
- 스레드 수 = CPU 코어 수 + 1

### I/O 바운드 작업

- 디스크, 네트워크, 파일 시스템 같은 입출력을 요구하는 스레드 작업
- I/O 작업 완료까지 대기 시간이 많아짐 → 유휴 상태의 CPU가 상대적으로 많아짐
- 스레드 수 > CPU 코어 수

### 웹 애플리케이션 서버

- 일반적으로 I/O 바운드 작업이 훨씬 많다
- 동시에 여러 요청이 들어오면 그 요청의 갯수만큼 스레드 갯수가 필요
- 대신 CPU 코어는 대기 시간이 많아지므로 스레드와 갯수를 꼭 맞출 필요는 없다
